Overview

1. Video representation

We use 3D CNN to extract feature for representing video. Previously, people use low-level such like HOG, HOF, MBH, IDTF, etc. Recently, researchers demonstrate the effectiveness of 3D CNN for representating video. 3D CNN can capture appreance and motion infomation simultaneously. For each video, we use 3D CNN model pre-trained on Sports1M dataset and then fine-tuned on UCF101 dataset, to extract FC7 activations as feature vector for describing videos in UCF101 dataset.

2. Hash

We use LSH to learn a hashing function on UCF101 dataset based on the video feature vector extracted by 3D CNN. For each video, we generate a binary code vector of 256 dimensionality.

3. Retrieval

A video retrieval system and visualization interface are developed. We use hamming distance of the hash code to measure similarity of two videos. Given a query video, our system identifies its 6 nearest neighbors which are relevant videos to be returned and shown to user.
